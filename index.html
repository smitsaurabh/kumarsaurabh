RAG MySQL Bot — Schema-Aware Natural Language DB Query

GitHub ↗

A schema-aware RAG chatbot enabling natural language querying over MySQL databases. Uses semantic search over DB schemas to generate accurate SQL, falls back to vector search or live web search when structured data is insufficient, and auto-adapts to schema changes — all powered by a local LLM with no cloud dependency.

  • Query Classification — LLM-driven classifier (with regex fallback) routes each question to the right tool: SQL, vector search, or web search.
  • Schema Layer — DB schema is introspected, embedded, and stored in a vector store; semantic search retrieves the most relevant tables before SQL generation.
  • Text-to-SQL — Local LLM generates SQL from relevant tables with EXPLAIN-based validation and one-retry self-correction on errors.
  • Row-Level Vector Store — Text columns from MySQL rows are chunked and embedded for semantic similarity search when SQL returns no results.
  • Web Search Fallback — Provider-agnostic layer (Perplexity → Tavily → DuckDuckGo) handles questions outside the database.
  • CDC Pipeline — Change Data Capture (polling) keeps the vector store in sync with live DB changes automatically.
  • Schema Watcher — Detects ALTER / CREATE / DROP TABLE events and re-indexes only the affected tables.
  • Chat API + UI — FastAPI server with SQLite-backed conversation history and a browser-based chat interface.
Python LangGraph LangChain FastAPI MySQL SQLite Vector Store Text-to-SQL RAG CDC Pipeline Local LLM Perplexity / Tavily / DuckDuckGo
